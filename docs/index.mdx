## Quickstart

<Steps>
  <Step title="Install @panther/core">
    <CodeGroup>
      ```bash pnpm theme={null}
      pnpm add @panther/core
      ```

      ```bash npm theme={null}
      npm i @panther/core
      ```
    </CodeGroup>

    This gives you the Panther SDK and the built-in `StdioTransport`, which can spawn MCP servers like the official filesystem server.
  </Step>

  <Step title="Define MCP servers">
    ```ts theme={null}
    import { McpServer, StdioTransport } from "@panther/core";

    const filesystem = new McpServer({
      name: "filesystem",
      displayName: "Filesystem",
      transport: new StdioTransport({
        command: "npx",
        args: ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
      }),
    });
    ```

    The `McpServer` wrapper lets Panther enumerate tools, prefix names to avoid collisions, and inject per-user env variables.
  </Step>

  <Step title="Start the proxy">
    ```ts theme={null}
    import { McpProxy } from "@panther/core";

    const proxy = new McpProxy({
      servers: [filesystem],
      port: 3000,
      path: "/mcp",
    });

    await proxy.start();
    ```

    You now expose a single MCP endpoint that routes to multiple upstream servers and enforces centralized policy.
  </Step>
</Steps>
